  • 为了捕获失败，异常的细节信息应该包含所有“对该异常有贡献”的参数和域的值

64. 努力使失败保持原子性

失败原子性：一般而言，失败的方法调用应该使对象保持在被调用之前的状态

实现失败原子性的几种途径：

  1. 设计一个不可变的对象
  2. 对于可变对象，在执行操作之前检查参数的有效性，使得在对象状态被修改之前，先抛出适当的异常
  3. 编写一段恢复代码（远没有那么常用）
